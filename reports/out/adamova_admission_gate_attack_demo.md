# Отчёт: результат диагностики кручения рациональных кривых в криптографическом контуре Kraken

Статус: evidence первичной реализации.
Дата: 2026-06-05.
Область: Android/Kotlin политика пакетов и сессий, опирающаяся на нативную
C++ диагностику кручения рациональных кривых.

## Назначение

Этот отчёт фиксирует не “режим исследования” сам по себе, а результат работы
диагностики кручения рациональных кривых внутри криптографического контура Kraken.

Диагностика кручения рациональных кривых используется как контур допуска экспериментального
криптографического профиля. Он не заменяет стандартное шифрование, подписи,
AEAD, Android Keystore или проверенный протокол. Его роль уже иная и более
узкая:

> не допустить слабые, сингулярные, подменённые или требующие эталонной
> проверки параметры экспериментального профиля до создания сессии и до
> отправки/приёма packet.

## Реализованный путь решения

```text
Криптографический профиль Kraken
  -> контур допуска криптографического профиля
  -> нативная C++ диагностика кручения рациональных кривых
  -> решение допуска
  -> metadata relationship/session/packet
  -> проверка при QR-сопряжении
  -> enforcement в outbox/inbox
```

Стандартный профиль:

```text
STANDARD_REVIEWED_PRIMITIVES
  -> диагностика кручения рациональных кривых не применяется
  -> профиль не блокируется этим контуром
```

Экспериментальный профиль:

```text
EXPERIMENTAL_ADAMOVA_CURVE_PROFILE
  -> C++ контур допуска диагностики кручения рациональных кривых обязателен
```

## Сценарии атаки и результат алгоритма

| Сценарий | Риск без контура допуска | Результат диагностики кручения рациональных кривых | Evidence |
| --- | --- | --- | --- |
| Сингулярная кривая | Наивный импорт мог бы принять некорректный профиль | Отклонить как сингулярную (`REJECT_SINGULAR`) | Unit test scenario |
| Рациональное 2-кручение | Профиль с малой структурой мог бы стать session profile | Отклонить как риск малой структуры (`REJECT_SMALL_TORSION_RISK`) | Unit test scenario |
| Индикатор 3-кручения | Подозрительный профиль мог бы пройти только по дискриминанту | Отклонить как риск малой структуры (`REJECT_SMALL_TORSION_RISK`) | Unit test scenario |
| Профиль вне локального бюджета | Дорогие/неподдержанные параметры могли бы быть ошибочно приняты | Ограничить по размеру/сложности (`SIZE_GUARDED`), без автоматического допуска | Unit test scenario |
| Некорректный профиль | Неверные коэффициенты могли бы обойти слабый importer | Требовать эталонную проверку или отклонить до активации | Unit test scenario |
| Понижение до слабого профиля | Peer мог бы заменить сохранённый профиль более слабым | Отклонить слабый профиль | Unit test scenario |
| Недоступный native backend | Экспериментальный профиль мог бы fail-open | Закрыть без допуска (`NATIVE_UNAVAILABLE`) | Unit test scenario |
| Неизвестный профиль в handshake | Contact мог бы стать `ACTIVE` с непроверенным профилем | Handshake возвращает ошибку; relationship не активируется | Unit test scenario |
| Неизвестный профиль в outbox | Message мог бы быть поставлен в очередь с непроверенным профилем | `CRYPTO_PROFILE_REJECTED`; packet не отправляется | Unit test scenario |
| Несовпадение профиля в packet | Входящий packet мог бы заявить другой профиль | `CRYPTO_PROFILE_MISMATCH`; сообщение не создаётся | Unit test scenario |

## Метрики первичного unit-backed snapshot

`AdamovaAdmissionAttackDemoRunnerTest` проверяет детерминированный набор
сценариев с fake native validator:

```text
profiles_total = 7
weak_profiles_total = 7
accepted_without_precheck = 6
accepted_by_discriminant_only = 5
accepted_by_adamova_gate = 0
rejected_by_adamova_gate = 7
needs_reference_validation = 1
size_guarded = 1
```

В русской интерпретации:

- всего сценариев профилей: 7;
- слабых или некорректных профилей: 7;
- принято без предварительной проверки: 6;
- принято при проверке только дискриминанта: 5;
- автоматически принято после диагностики кручения рациональных кривых: 0;
- отклонено или заблокировано диагностикой кручения рациональных кривых: 7;
- требует эталонной проверки: 1;
- ограничено по размеру/сложности: 1.

Эти числа подтверждают поведение политики допуска в Android code path. Они не
являются live phone metrics, SageMath reference results или доказательством
промышленной криптостойкости сообщений.

## Разрешённое утверждение

Диагностика кручения рациональных кривых встроена в Kraken как нативный C++ контур допуска
экспериментального криптографического профиля. В контролируемой модели она
предотвращает попадание слабых, сингулярных, подменённых или локально
непроверяемых параметров в relationship/session/packet policy.

## Граница утверждения

Этот отчёт не доказывает промышленную криптографическую безопасность сообщений
Kraken и не заменяет проверенные примитивы, подписи, AEAD, защиту от replay,
Android Keystore и внешний security review.

# Quiet Graphite Product UI Reference

Saved from the user-provided Kraken Messenger reference image.

## Target Direction

- Messenger-first UI, not a research dashboard.
- Dark graphite background, not pure black.
- Calm teal/cyan accent, low neon.
- Compact readable rows over oversized debug cards.
- Research features live in Settings/Research mode.
- Technical IDs and certificates stay hidden under technical details.
- Primary navigation: Chats, Contacts, Realms, Settings.

## Screen Patterns

- Overview: compact identity block, "at a glance" list, 3 compact quick actions, small research shortcut.
- Contacts: active, pending handshake, blocked/unlinked sections with dense rows and always-visible state badges.
- Chat: message-first layout, compact peer header, composer at bottom, no QR shortcuts.
- Realms: grouped active/pending/left sections with compact membership cards.
- Settings: privacy/control-center sections, no experimental debug copy in the first viewport.
- My QR: QR-first share screen with lifecycle controls below.
- Scanner: focused camera frame with short copy and manual fallback.

## Production Rules

- Russian is the default user language for production flow.
- No top-right Back text button in production screens.
- No production crypto claims.
- No account, phone, email, public discovery, P2P, or networking claims.
- Copy should be short and product-oriented; caveats remain but do not dominate primary UI.

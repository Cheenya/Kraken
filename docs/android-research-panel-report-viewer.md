# Android Research Panel Report Viewer

Date: 2026-05-19

This change integrates a bundled math-core curve diagnostic report into the Android Research panel.

## What was integrated

- Android-compatible report contract:
  `kraken.math.curve_diagnostic.android.v1`
- Bundled sample report:
  `app-android/app/src/main/assets/research/sample_curve_diagnostic_report.json`
- Kotlin report models:
  `app-android/app/src/main/java/com/disser/kraken/research/CurveDiagnosticReport.kt`
- Asset/parser repository:
  `app-android/app/src/main/java/com/disser/kraken/research/CurveReportRepository.kt`
- Research screen display cards for the bundled report.

## Source report

The bundled asset is copied from:

```text
reports/examples/sample_curve_diagnostic_report.json
```

That report is generated by the Python math-core Android export adapter on the
math integration branch. Android does not generate it and does not run Python.

## How Android reads the report

`CurveReportRepository.loadBundledSampleReport(context)` reads:

```text
research/sample_curve_diagnostic_report.json
```

from app assets, parses it with `kotlinx.serialization`, validates:

```text
report_version == kraken.math.curve_diagnostic.android.v1
```

and returns a success/error state.

There is no network, server, cloud sync, account dependency, or Python runtime.

## Research panel display

The Research screen now shows:

- diagnostic-only warning;
- curve equation;
- discriminant;
- singular status;
- j-invariant;
- 2-torsion summary;
- Lutz-Nagell candidate summary;
- bounded torsion probing summary;
- benchmark summary;
- warnings and unsupported cases.

Required wording remains visible:

```text
Diagnostic-only. This is not production encryption and does not prove Kraken message security.
```

## Limitations

- The report is static bundled demo data.
- The report is not production encryption.
- The report does not prove Kraken message security.
- Lutz-Nagell output is candidate diagnostics, not a complete torsion proof.
- Torsion probing is bounded.
- Future interactive Android diagnostics require a C++/JNI migration with parity tests against Python.

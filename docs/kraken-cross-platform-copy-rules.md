# Kraken Cross-Platform Copy Rules

Правила для Android, iOS, macOS и Windows-версий Kraken. Цель документа —
не дать старым рабочим формулировкам снова попасть в пользовательский UI,
README и новые evidence-отчёты.

## Базовый тон

- Пользовательский текст пишем по-русски.
- Английскими оставляем только устойчивые имена технологий и продуктов:
  Kraken, QR, BLE, LAN, Wi-Fi Direct, Android, iOS, macOS, Windows,
  SwiftUI, PySide6, Xcode, Python, JSON, ADB.
- Технические JSON-ключи, enum raw values, UUID, CLI flags и test fixtures не
  переводим, если они являются частью протокола или проверки.
- В обычном UI не показываем сырые `profile_id`, `decision_hash`,
  `transport_id`, `relationship_id`, `payload_json`, enum `.value` и risk flags.
  Их можно показывать в раскрываемом блоке `Технические данные`, diagnostics
  или evidence export.

## Словарь

- Пишем `Профиль Kraken`, а не `локальная личность`.
- Пишем `локально`, `локальная связь`, `локальное устройство`, а не слоган
  `рядом` или корявое `связь рядом`.
- Пишем `Реалмы`, а не `рилмы`.
- Для QR используем:
  - `QR-приглашение`;
  - `QR-ответ`;
  - `QR подтверждения`.
- Не возвращаем `ответный QR`, `финальный QR`, `резервный QR` как основные
  названия экранов или действий.
- Для криптографии пишем `контур допуска криптографического профиля`,
  `привязка контекста`, `точки кручения порядка 2/3`.
- Не пишем `алгоритм Адамовой` в пользовательском UI и публичных кратких
  описаниях. Если нужно назвать реализацию, используем `Adamova` как имя
  внутреннего профиля или research-модуля.

## Границы утверждений

- Не используем самоуничижительные формулировки в стиле `not production`,
  `no security`, `no E2EE`, `не продакшен-мессенджер`.
- Вместо этого описываем, что конкретно проверяет версия:
  `Android Wi-Fi Direct/BLE`, `macOS LAN/ADB-мост`, `Windows LAN/BLE/QR`,
  `iOS MultipeerConnectivity`, `evidence export`, `контур допуска профиля`.
- Если артефакт чего-то не закрывает, пишем предметно:
  `доставка по Android-радиоканалу подтверждается отдельным телефонным прогоном`.

## Документация версий

- `app-android/` остаётся основным источником поведения Android, QR/nearby
  сопряжения и Wi-Fi Direct/BLE evidence.
- `app-ios/` использует нативный iOS UI и Apple MultipeerConnectivity для
  локальной связи; не описывать это как Android Wi-Fi Direct.
- `app-macos/` проверяет desktop-сценарии, LAN/ADB-мост, CoreBluetooth и
  совместимость форматов с Android.
- `app-windows/` проверяет Windows desktop UI, LAN/BLE/QR-форматы и локальное
  состояние; нативный Windows Wi-Fi Direct не реализуется.

## Проверка перед завершением

Перед финалом прогоняем grep по активным версиям:

```bash
rg -n "локальная личность|Локальная личность|ответный QR|финальный QR|резервный QR|связь рядом|not production|no E2EE|no security|production-secure|алгоритм Адамовой" app-ios app-macos app-windows docs
```

Ожидаем только технические guard-тесты, исторические журналы или этот документ.
